## 2. 当前版本概览

### 2.1 原型定位
`web-test` 不是生产代码，而是一套高保真静态原型。它以假登录和本地测试数据驱动页面，用来验证“时光机器”在 Web 端的核心信息架构、叙事结构和页面原型设计。

### 2.2 本次已落地内容
当前版本相对于早期单页原型，已经完成以下升级：

- 增加登录入口页，模拟产品入口和会话状态。
- 从单页切换升级为多页面原型。
- 增加事件详情页、地点详情页、人物详情页。
- 把回忆录视图升级为更细的编辑器形态。
- 继续使用静态测试数据，不依赖后端。
- 用 `localStorage` 模拟登录态和页面工作台状态。

### 2.3 本次重点验证的问题

- 用户是否能先通过登录入口理解产品入口形态。
- “时间、空间、人物、回忆录”四个一级入口是否清晰。
- 事件、地点、人物是否已经形成真正的多页面浏览体系。
- 回忆录是否不再只是章节列表，而是接近编辑器工作台。
- 在静态测试数据下，是否已经能清楚表达产品的核心价值。

### 2.4 当前不覆盖范围

- 真实账号系统
- 真实上传、导入、实时采集
- 真实地图服务
- 真实音视频播放
- 真实回忆录保存与发布
- 可点击的移动端独立原型页
- React/Vite 正式工程骨架

### 3.1 页面入口文件

- `index.html`
  - 只负责判断是否已有会话。
  - 已登录时跳转 `timeline.html`。
  - 未登录时跳转 `login.html`。

- `login.html`
  - 登录页入口。
  - 承载邮箱密码登录和一键测试账号登录。

- `timeline.html`
  - 时间线总览页。
  - 作为默认进入后的首页。

- `space.html`
  - 空间总览页。
  - 强调地点是独立浏览入口。

- `people.html`
  - 人物总览页。
  - 强调关系不是联系人，而是共同经历索引。

- `memoir.html`
  - 回忆录编辑器页。
  - 强调章节树、正文区和候选事件库同时存在。

- `event.html`
  - 事件详情页。
  - 连接时间、空间、人物和回忆录。

- `place.html`
  - 地点详情页。
  - 把地点组织成“空间档案”。

- `person.html`
  - 人物详情页。
  - 把人物组织成“共同记忆档案”。

### 3.2 数据与运行文件

- `data.js`
  - 定义全部静态测试数据。
  - 定义页面默认状态。
  - 提供简易数据查询方法。

- `prototype.js`
  - 管理登录态和页面状态。
  - 渲染各页面视图。
  - 处理筛选、跳转、详情打开和退出登录。

- `styles.css`
  - 提供视觉语言、布局系统、卡片体系和响应式规则。

## 5. 信息架构

### 5.1 一级结构
登录后进入主工作台，一级导航固定为：

- `时间线`
- `空间`
- `人物`
- `回忆录`

### 5.2 详情页结构
在一级视图之外，原型增加三个二级详情页：

- `事件详情页`
- `地点详情页`
- `人物详情页`

这三页的意义是把原来只在单页内局部展开的内容，升级为可单独承载叙事和索引关系的独立页面。

### 5.3 页面跳转主链路
当前主链路已经是可点击闭环：

1. 登录页
2. 时间线页
3. 事件详情页
4. 地点详情页 / 人物详情页
5. 回忆录页

同时支持横向切换：

- 时间线 -> 空间
- 时间线 -> 人物
- 空间 -> 事件详情
- 人物 -> 回忆录
- 事件详情 -> 地点详情 / 人物详情 / 回忆录

## 6. 页面设计说明

### 6.1 登录页
登录页不只是表单，而是产品入口页。

主要目标：

- 让用户在进入原型前就理解产品价值。
- 模拟正式产品的登录入口。
- 提供不同类型用户的演示入口。

主要区域：

- 品牌区
- 产品价值说明
- 三个能力摘要卡片
- 邮箱密码登录表单
- 一键登录测试账号卡片

关键交互：

- 手动输入测试账号密码进入原型
- 点击演示账号一键进入原型
- 已登录时访问登录页会自动跳回时间线页

### 6.2 时间线页
时间线页是登录后的默认首页，强调“事件优先”。

主要区域：

- 顶部导航与搜索
- 左侧年份筛选和阶段筛选
- 中央当前选中事件 Hero
- 时间线事件列表
- 右侧地图、人物网络、摘要层

关键交互：

- 搜索事件、地点、人物
- 选择年份
- 选择人生阶段
- 打开事件详情页
- 从当前事件跳到地点页或人物页

### 6.3 空间页
空间页强调地点不是标签，而是独立视角。

主要区域：

- 当前地点主叙事
- 地图节点切换区
- 地点相关事件流
- 右侧地点速览

关键交互：

- 点击地图节点切换地点
- 打开该地点代表事件
- 进入地点详情页
- 从地点回到事件或跳到人物

### 6.4 人物页
人物页强调“关系即共同经历索引”。

主要区域：

- 当前人物主叙事
- 关系网络图
- 共同事件流
- 共同地点
- 可写成回忆录章节的片段

关键交互：

- 切换当前人物
- 打开人物详情页
- 把人物相关内容导向回忆录

### 6.5 回忆录页
回忆录页已经从“展示章节”升级为“编辑器原型”。

主要区域：

- 章节树
- 编辑器正文区
- 章节封面草稿
- 叙事片段块
- 媒体布局区
- 候选事件库
- 当前章节状态

这一页重点体现：

- 章节是可以编辑的工作对象。
- 事件不是展示终点，而是写作素材来源。
- 时间、空间、人物可以被重新编排为叙事章节。

### 6.6 事件详情页
事件详情页负责把单个事件做成立体叙事页面。

主要区域：

- 事件主叙事
- 时间、地点、人物、素材总数
- 地点与人物索引区
- 相关事件流
- 右侧地图和关系摘要

它是多页面结构中的关键中枢页。

### 6.7 地点详情页
地点详情页强调“地点档案”。

主要区域：

- 地点说明和事实信息
- 地图节点网络
- 地点时间切片
- 右侧地点速览

它让空间维度从总览页面延伸到完整档案页。

### 6.8 人物详情页
人物详情页强调“共同记忆档案”。

主要区域：

- 人物关系说明
- 关系图
- 人物关键事件
- 共同地点
- 适合写成一章的片段

它让人物维度具备独立的信息承载能力。

## 7. 设计原则

当前原型持续遵循以下页面原型设计原则：

- 默认以事件为核心组织内容，而不是平铺素材。
- 时间、空间、人物是并列入口，不互相从属。
- 详情页不是弹层，而是独立页面。
- 回忆录不是列表页，而是工作台。
- Web 端重点突出整理、回看、编排，而不是即时拍摄。
- 所有测试数据都尽量围绕“成长录 / 回忆录”叙事来组织。
