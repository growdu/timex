# 时光机器设计文档

## 1. 文档说明

## 2. 设计目标

### 2.1 体验目标

- 低门槛记录：任何一个瞬间都可以快速进入系统。
- 高密度回顾：用户能从大量素材中迅速建立“这段时间发生了什么”的理解。
- 立体化记忆：同一份内容能够从时间、空间、人物三个角度被重新组织。
- 长期可沉淀：素材最终不只是被存储，而是被讲述、被编排、被回看。

### 2.2 核心体验判断标准
一个设计方案是否成立，可以用以下标准检验：

- 用户进入首页后，3 秒内能理解“这是我的人生时间线”。
- 用户打开任意事件后，能快速看到时间、地点、人物、内容四类关键信息。
- 用户从地图或人物入口进入后，不会迷失，能顺畅回到事件和时间流。
- 用户在移动端完成记录，在 Web 端完成整理时，不会感到两套产品割裂。

## 3. 设计原则

### 3.1 时间是主骨架
所有内容最终都要挂在时间之上。即使用户从地点或人物进入，也必须能返回时间轴。

### 3.2 地点和人物不是标签，而是入口
地点和人物不应只是事件上的附属字段，而应成为独立浏览入口和独立详情页。

### 3.3 事件优先于原始素材
默认展示单位应为事件，而不是图片列表。事件是用户理解和回忆的最小叙事单元。

### 3.4 先读懂，再深入
先给用户总结和结构，再给素材细节。先展示标题、摘要、地点、人物，再展示全部图片、视频、语音和文字。

### 3.5 统一结构，分端表达
移动端和 Web 端使用同一套信息模型，但通过不同布局表达。不要做两套逻辑不同的产品。

## 4. 体验架构

### 4.1 三维体验模型
时光机器的整体体验应被理解为三维记忆系统：

- 时间维度：回答“什么时候发生了什么”。
- 空间维度：回答“在哪里发生了什么”。
- 人物维度：回答“和谁发生了什么”。

三维模型最终通过“事件”汇合。事件是三维信息的交点。

### 4.2 内容层级

- 人生档案：用户的全部记忆集合。
- 阶段：一段较长周期的人生章节。
- 事件：一组连续相关的记录。
- 瞬间：一次记录动作产生的素材集合。
- 素材：图片、视频、语音、文字的原始内容。

### 4.3 页面组织逻辑

- 首页承担时间主入口。
- 空间页承担地图主入口。
- 人物页承担关系主入口。
- 事件详情页承担信息汇聚与叙事展开。
- 回忆录页承担长期沉淀和输出。

## 5. 导航设计

### 5.1 移动端主导航
移动端采用底部五栏导航：

- 时间线
- 空间
- 人物
- 记录
- 我的

说明：

- `记录` 为高频动作入口，可采用强调按钮样式。
- `导入` 建议收进 `记录` 页二级入口，而不是常驻底部导航。
- `回忆录` 建议先放入时间线页和我的页中，避免底部导航过重。

### 5.2 Web 端主导航
Web 端采用左侧主导航 + 顶部操作栏：

- 时间线
- 空间
- 人物
- 导入中心
- 回忆录
- 我的档案

顶部操作栏统一承载：

- 全局搜索
- 新建记录
- 筛选器
- 账号入口

### 5.3 二级导航规则
所有一级入口内部都建议提供统一二级切换逻辑：

- 视图切换：列表、地图、关系图、阶段
- 时间范围：全部、近 7 天、近 30 天、今年、自定义
- 内容类型：图片、视频、语音、文字、混合

## 6. 视觉方向

### 6.1 整体气质
设计风格应避免通用相册产品的工具感，也避免纯社交产品的热闹感。整体应呈现：

- 温和
- 可信
- 沉浸
- 有记忆感

### 6.2 视觉关键词

- 时间轴
- 档案感
- 场景感
- 人物关系
- 留白
- 层次清晰

### 6.3 色彩建议

- 主色：偏暖中性色，表达记忆与陪伴感。
- 辅色：用于时间、地点、人物三类信息区分。
- 强调色：仅用于记录按钮、当前状态和重点 CTA。

建议：

- 时间维度可偏深蓝或墨色。
- 空间维度可偏青绿。
- 人物维度可偏暖橙或土红。
- 背景采用柔和浅色，避免纯黑纯白造成工具压迫感。

### 6.4 字体与排版建议

- 标题层级明确，帮助用户快速扫读事件。
- 正文排版适合长文本与旁白阅读。
- 时间、地点、人物字段要有稳定视觉位置。

### 6.5 动效原则

- 事件展开应平滑，增强“翻阅回忆”的感受。
- 地图和时间线联动应有清晰反馈。
- 人物关系图切换不应炫技，重点是可理解。

## 7. 核心组件设计

### 7.1 事件卡片
事件卡片是全产品最核心组件，默认承担列表浏览入口。

#### 标准字段

- 封面图
- 事件标题
- 时间范围
- 地点信息
- 人物摘要
- 内容摘要
- 素材统计

#### 组件变体

- 标准事件卡：用于首页时间线。
- 紧凑事件卡：用于地点详情和人物详情侧栏。
- 大图事件卡：用于回忆录和年度回顾。

#### 交互要求

- 点击进入事件详情。
- 支持长按或悬停唤起快捷操作。
- 快捷操作包括编辑、收藏、加入回忆录、分享。

### 7.2 人物卡片

#### 标准字段

- 头像或代表图
- 姓名/称谓
- 关系标签
- 共现次数
- 最近共同记录时间

#### 交互要求

- 点击进入人物详情页。
- 支持按关系筛选。
- 支持从事件详情中快速添加或修正人物关联。

### 7.3 地点卡片

#### 标准字段

- 地点名称
- 地点类型
- 首次出现时间
- 最近出现时间
- 关联事件数

#### 交互要求

- 点击进入地点详情页。
- 支持在地图页和搜索页复用。

### 7.4 素材块
素材块是事件详情页中的基础展示单元。

#### 图片块

- 支持单图、大图、多图拼贴三种样式。
- 支持查看原图、滑动切换、缩放。

#### 视频块

- 显示封面、时长、播放入口。
- 默认不自动播放。

#### 语音块

- 显示时长、波形、转写摘要。
- 点击可播放和展开全文转写。

#### 文字块

- 展示标题、正文摘要、展开全文。
- 可标记为日记、备忘、旁白三种类型。

### 7.5 筛选器组件

- 时间筛选
- 地点筛选
- 人物筛选
- 内容类型筛选
- 阶段筛选

统一要求：

- 筛选器状态必须可见。
- 支持一键清空。
- Web 端支持多条件同时展示。

## 8. 页面设计规格

### 8.1 移动端首页时间线页

#### 页面目标
让用户以最自然的方式回看时间，并快速进入记录。

#### 页面结构

- 顶部栏：日期切换、搜索、消息或提醒入口。
- 主内容区：按时间顺序排列的事件卡片流。
- 浮动按钮：快速记录。
- 底部导航：五栏主导航。

#### 关键模块

- 今日卡片：展示当天记录和补记入口。
- 本周回顾：展示近 7 天高频事件。
- 年份锚点：帮助快速跳转到某一年或某个月。
- 阶段入口：在时间线中露出重要阶段节点。

#### 关键交互

- 上下滚动浏览时间。
- 点击年份锚点快速定位。
- 下拉进入时间筛选器。
- 左右滑动事件卡可显示快捷操作。

### 8.2 移动端空间页

#### 页面目标
帮助用户从地点理解自己的经历分布。

#### 页面结构

- 顶部：搜索框、筛选按钮。
- 中部：可交互地图。
- 下部：当前视口或当前地点下的事件流。

#### 关键模块

- 地点热区
- 旅行路线切换
- 地点详情入口
- 时间滑杆

#### 关键交互

- 点击地图点位查看地点摘要。
- 拖动地图刷新下方事件流。
- 切换年份后地图热区重新计算。

### 8.3 移动端人物页

#### 页面目标
让用户从人与关系出发回顾共同记忆。

#### 页面结构

- 顶部：搜索、关系分类标签。
- 主体：人物卡片列表或关系图简版。
- 底部：当前人物相关事件预览。

#### 关键模块

- 关系分组：家人、朋友、同学、同事、伴侣。
- 常见共现人物：按共现频次排序。
- 关系时间线：展示关系变化节点。

#### 关键交互

- 点击人物进入人物详情。
- 切换关系分类刷新列表。
- 点击共现人物进入双人共同记忆集合。

### 8.4 移动端记录页

#### 页面目标
降低创建门槛，使记录行为足够轻。

#### 页面结构

- 顶部：返回、草稿入口。
- 主体：四类主按钮，图片、视频、语音、文字。
- 次级区：导入、实时采集、补记、扫一扫。

#### 关键交互

- 进入页面 1 次点击即可开始记录。
- 记录完成后默认进入补充信息页。
- 支持直接保存为草稿。

### 8.5 Web 端时间线页

#### 页面目标
支持更高效率的浏览、筛选与批量整理。

#### 页面布局

- 左侧：年份/阶段导航树。
- 中央：事件列表。
- 右侧：事件预览面板或筛选器面板。

#### 关键能力

- 批量选择事件。
- 快速修改标题、标签、阶段归属。
- 批量加入回忆录章节。

### 8.6 Web 端空间页

#### 页面目标
以地图为中心进行空间回顾和编辑。

#### 页面布局

- 左侧：地点列表和搜索筛选。
- 中央：地图视图。
- 右侧：当前地点时间线。

#### 关键能力

- 批量修正地点信息。
- 拖动时间范围联动地图点位。
- 查看不同年份同一地点的变化。

### 8.7 Web 端人物页

#### 页面目标
以人物关系为中心进行整理和叙事挖掘。

#### 页面布局

- 左侧：人物列表与关系分类。
- 中央：人物关系图或人物详情。
- 右侧：与当前人物相关的事件和地点。

#### 关键能力

- 合并重复人物。
- 修正人物关系标签。
- 生成“与此人相关的回忆录片段”。

### 8.8 事件详情页

#### 页面目标
把一段记忆完整讲述出来。

#### 页面结构

- 头图区：封面、标题、时间范围。
- 摘要区：自动摘要和用户旁白。
- 地点区：地图缩略图、地点信息、历史次数。
- 人物区：人物卡片、关系标签、共现关系。
- 素材区：按时间顺序混排图片、视频、语音、文字。
- 关联区：同地点、同人物、同阶段的相关推荐。

#### 关键交互

- 支持素材排序调整。
- 支持补充地点和人物关联。
- 支持加入某一阶段或回忆录章节。

### 8.9 地点详情页

#### 页面目标
围绕地点构建“场所记忆”。

#### 页面结构

- 顶部：地点名称、地图、地点类型。
- 中部：该地点的时间线。
- 下部：与该地点相关的人物和代表事件。

#### 关键交互

- 支持按年份筛选地点记忆。
- 支持查看同地点不同阶段的变化。

### 8.10 人物详情页

#### 页面目标
围绕人物构建“共同记忆”。

#### 页面结构

- 顶部：人物头像、关系、第一次出现时间、最近出现时间。
- 中部：相关事件时间线。
- 侧边或下方：共同去过的地点、共同素材精选。

#### 关键交互

- 支持查看与该人物的阶段变化。
- 支持切换双人、多人的共同记忆视角。

### 8.11 阶段页

#### 页面目标
帮助用户从碎片事件上升到人生阶段理解。

#### 页面结构

- 阶段头图
- 阶段简介
- 关键人物
- 关键地点
- 阶段时间线
- 代表素材

#### 关键交互

- 支持拖拽事件调整阶段归属。
- 支持生成阶段摘要。

### 8.12 回忆录编辑页

#### 页面目标
把事件集合编排为可读、可分享的长期内容。

#### 页面结构

- 左侧：章节目录。
- 中央：正文编辑区。
- 右侧：素材库、事件库、人物库、地点库。

#### 关键能力

- 章节增删改排序。
- 事件一键插入章节。
- 图片、视频、语音、文字混排。
- 自动生成目录和阶段摘要。

### 8.13 搜索页

#### 页面目标
统一承接所有检索需求。

#### 页面能力

- 支持关键词搜索。
- 支持按时间、地点、人物、类型组合筛选。
- 搜索结果可切换为事件、地点、人物、素材四种视图。

## 9. 关键用户流程

### 9.1 日常记录流程

1. 用户点击 `记录`。
2. 选择图片、视频、语音或文字。
3. 完成采集后补充标题、地点、人物。
4. 内容进入时间线，并被自动归并到事件中。

### 9.2 历史导入流程

1. 用户进入导入中心。
2. 选择导入来源。
3. 系统解析时间、地点、内容信息。
4. 用户确认导入结果和自动分组。
5. 导入后的内容进入待整理列表或直接进入时间线。

### 9.3 从人物回看共同记忆流程

1. 用户进入人物页。
2. 选择某个人物。
3. 查看与该人物的事件时间线。
4. 进一步切换到地点分布或阶段视图。
5. 将相关事件加入回忆录章节。

### 9.4 从地点回看人生轨迹流程

1. 用户进入空间页。
2. 点击某个城市或地点。
3. 查看该地点下的全部事件。
4. 切换年份查看不同时期。
5. 进入代表事件详情完成深度回看。

## 10. 状态设计

### 10.1 空状态

- 无记录：引导用户创建第一条记录。
- 无地点信息：引导补充位置信息或授权定位。
- 无人物信息：引导添加人物标签。
- 无搜索结果：提供相关筛选建议。

### 10.2 加载状态

- 时间线骨架屏
- 地图点位加载态
- 人物卡片骨架屏
- 回忆录编辑器保存中状态

### 10.3 错误状态

- 上传失败
- 导入解析失败
- 地图加载失败
- 同步失败

所有错误都应提供：

- 明确原因
- 重试入口
- 不丢失当前上下文

### 10.4 离线状态

- 支持离线创建图片、语音、文字记录。
- 对未同步内容给出明确标记。
- 联网后自动续传并同步。

## 11. 智能能力在设计中的落点

### 11.1 自动摘要
系统可为事件生成一句话摘要，用于首页卡片和事件详情页头部。

### 11.2 自动标题
系统根据时间、地点、人物、内容生成事件标题，用户可手动修改。

### 11.3 自动聚类
系统把连续记录自动归并为事件，并在设计上用“待确认事件”提示用户快速确认。

### 11.4 自动人物和地点识别
系统可推荐地点与人物关联，但界面必须保留人工修正入口。

### 11.5 自动回忆录草稿
系统根据阶段或主题自动生成章节目录和摘要，用户负责最后编排。

## 12. 双端差异化策略

### 12.1 移动端职责

- 即时记录
- 快速浏览
- 轻量补记
- 情绪化回顾

### 12.2 Web 端职责

- 批量导入
- 批量整理
- 深度筛选
- 回忆录编排

### 12.3 一致性要求

- 命名一致
- 信息结构一致
- 入口路径尽量一致
- 卡片字段一致

## 13. 可访问性与隐私设计

### 13.1 可访问性

- 关键文本需满足可读字号。
- 时间、地点、人物信息不能仅靠颜色区分。
- 图片和视频封面需支持替代文本策略。
- 交互控件在移动端需满足可点击尺寸要求。

### 13.2 隐私设计

- 默认私密。
- 公开分享需二次确认。
- 人物识别和地点识别需提示用户授权范围。
- 对孩子、家庭、敏感地点等内容应支持快速隐藏和权限控制。

## 14. 设计交付建议

### 14.1 交付物清单

- 信息架构图
- 移动端线框图
- Web 端线框图
- 事件卡片组件规范
- 人物卡片组件规范
- 地点卡片组件规范
- 交互流程图
- 视觉风格板

### 14.2 优先级建议

- 第一优先级：时间线页、事件详情页、记录页
- 第二优先级：空间页、人物页、导入中心
- 第三优先级：阶段页、回忆录编辑页、搜索页

## 15. 总结
时光机器的设计关键不在于做一个好看的相册，而在于把人生素材组织成一个可以被回顾、被理解、被讲述的三维记忆系统。设计必须始终围绕时间、空间、人物三个入口展开，并通过事件详情页完成信息汇聚，再通过阶段页和回忆录页完成长期沉淀。
